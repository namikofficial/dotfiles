# Tracked vs Generated Policy

## Tracked in Git

- Source config for Hyprland, Waybar, Rofi, SwayNC, lock/idle scripts.
- `settings/defaults.json`, `settings/schema.json`, and profile files.
- KDE defaults (`kde/`) and MIME defaults (`mime/mimeapps.list`).
- System profile templates under `system/`.

## Tracked Local Overrides

- `settings/state.json` is tracked for portable overrides.

## Local-Only (Not Tracked)

- `settings/state.local.json` (machine-specific and private local overrides).
- Runtime/generated files in `~/.cache/hypr/*`.
- Generated theme artifacts under app-specific config folders.

## Generated by Settings Apply

- `~/.cache/hypr/settings.generated.conf`
- `~/.cache/hypr/lock-settings.generated.conf`
- `~/.cache/hypr/app-routing.generated.json`
- other `~/.cache/hypr/*-settings.generated.json`

## Rule

If a value is user preference and should sync to git, put it in `state.json`.
If a value is machine-specific or experimental, keep it in `state.local.json`.
